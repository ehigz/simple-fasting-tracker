Guide

How to Use It

Building a Screen
1
Import from src/ui — never from primitives
All components, tokens, and scales export from the design system index.
import { Card, Button, CardTitle, BodyText, colors, typography } from "../ui";
2
Read the metadata files before creating anything new
src/ui/design-system.metadata.ts documents selectionCriteria and antiPatterns for every component.
3
NativeWind className for layout · StyleSheet for icon colors
className="gap-4 items-center" · color={`${colors.primary}`}
4
Use typography.scale.* for text outside the standard component set
const styles = StyleSheet.create({"{"} timer: {"{"} ...typography.scale.statHero {"}"} {"}"});
5
Run the audit before committing
The ESLint no-hardcoded-hex rule catches violations. The pre-commit hook runs it automatically.
npm run audit:tokens
Anti-patterns
NeverHardcode hex — use colors.* from theme.ts
NeverBare Pressable/TouchableOpacity — use Button
NeverBare Text — use a typography primitive
NeverEmoji as icons — use Lucide with colors.*
NeverBare View as card — use Card
NeverTwo primary buttons on one screen

Sync & Automation

How Automation Works

Claude Code is not a persistent daemon — it responds when invoked. The figma-console-mcp Desktop Bridge fills the Figma→Code gap by buffering changes in real-time. Here's how the five layers of automation keep things in sync.

Automation Layers
LIVE
🖥️ Styleguide File Watcher — always on while server runs
This dev server watches src/ui/theme.ts, design-tokens.json, and design-system-manifest.json. When any file changes, the dashboard updates in real-time via SSE — no refresh needed. Detects when theme.ts is newer than the last Figma sync and shows a warning banner.
cd styleguide && node server.js
SCHEDULED
⏱️ Claude Code Cron — drift check every 30 min while Claude is running
A Claude Code cron job runs /design-system-architect drift on a 30-minute interval. Results appear in the dashboard Action Items feed when the server detects an audit cache update. Note: this only runs while Claude Code is active in a session.
CronCreate: /design-system-architect drift (every 30m)
GIT HOOK
🪝 Pre-commit Hook — audit on every commit
A git pre-commit hook runs npm run audit:tokens (via audit_codebase.py) before every commit. Blocks commits with hardcoded hex values or bare View/Text/Pressable usage. Results cached to styleguide/.audit-cache.json for the dashboard.
.git/hooks/pre-commit → npm run audit:tokens
MANUAL
🔄 Code → Figma — run push after editing theme.ts
When you edit token values in theme.ts, the dashboard detects the drift and shows a warning. Run the push command to re-extract tokens and batch-update Figma variables. Takes ~5 seconds.
/design-system-architect push
MANUAL
⬇️ Figma → Code — run pull after designer updates Figma
When a designer updates variables or text styles in Figma, run pull. It fetches the full design system kit, generates a reviewed diff, and patches theme.ts and tailwind.config.js after your approval.
/design-system-architect pull
LIVE
🔌 Figma Desktop Bridge — MCP plugin buffers design changes in real-time
The figma-console-mcp Desktop Bridge plugin runs inside Figma Desktop and buffers every documentchange event over a WebSocket to Claude Code. The 30-min drift cron calls figma_get_design_changes { since, clear: true } — if token-relevant nodes (variables/styles) changed, it automatically triggers a Phase 3 pull. No webhook or external infrastructure needed. Requirement: Figma Desktop must be open with the plugin running.
figma_get_design_changes { since: lastCheckedAt, clear: true } → auto-pull if token changes
